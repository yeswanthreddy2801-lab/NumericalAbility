DATA INTERPRETATION — GRAPHS (DETAILED NOTES)

============================================================
1. PURPOSE & APPROACH

DI graphs present numerical data visually so you can quickly compare categories, spot trends, and answer quantitative questions under time pressure. Approach every graph question by (a) understanding the axes and units, (b) identifying totals or baselines, and (c) choosing the fastest computation route (direct read, percentage, ratio or difference).

============================================================
2. COMMON GRAPH TYPES & WHEN TO USE THEM

- Bar chart: compare discrete categories (sales per product, population per city).
- Stacked bar: compare component contributions to totals across categories.
- Line chart: track changes over time and spot trends.
- Pie chart: show proportion of a whole (useful when components sum to a known total).
- Histogram: frequency distribution of continuous data (bins on x-axis).
- Scatter plot: relationship between two continuous variables (correlation).
- Combo (line + bar): compare absolute values and trends simultaneously.

============================================================
3. KEY FORMULAS & CONCEPTS

- Percentage = (part / whole) × 100.
- Value from percentage = (percentage × total) / 100.
- Angle (pie) = percentage × 3.6.
- Ratio → percentage: a/(a+b+...) × 100.
- Average = sum / count.
- Rate of change = (new − old) / old × 100%.

Pro tip: work in percentages or fractions when totals are unknown; pick a convenient total (100 or LCM) for ratio problems.

============================================================
4. READING A GRAPH: SYSTEMATIC CHECKLIST

1) Title — confirms context.
2) Axes labels and units (thousands, millions, %).
3) Scale increments — note non-uniform scales.
4) Legend (colors/patterns) and grouped/stacked meanings.
5) Time range for trend graphs (start/end points).
6) Any footnotes or missing-data markers.

Skipping these can cause common mistakes; always scan them before computing.

============================================================
5. SOLVING STRATEGIES (BY QUESTION TYPE)

a) Exact value/query: read and compute using axis scale; verify with neighbors.
b) Percentage/share: compute part/whole; if total absent, assume 100 and find relative shares.
c) Difference/ratio: subtract or divide sector values; convert to percentage if needed.
d) Trend question: compute slope or percent change between two time points; check intermediate peaks/troughs.
e) Combined questions: normalize units (e.g., convert lakhs → thousands) before arithmetic.

============================================================
6. WORKED EXAMPLES

Example 1 — Bar chart read
Given bars showing yearly sales (2018: 20k, 2019: 30k, 2020: 45k, 2021: 40k). Q: Which year had the highest percentage increase over previous year?
- 2019 over 2018 = (30−20)/20 = 50%.
- 2020 over 2019 = (45−30)/30 = 50%.
- 2021 over 2020 = (40−45)/45 = −11.11%.
- Tie between 2019 and 2020 at 50%.

Example 2 — Stacked bar
If a stacked bar shows components A,B,C summing to 500 for category X and A=150,B=200,C=150; find percentage share of B in X = 200/500 ×100 = 40%.

Example 3 — Pie + total unknown
Pie sectors: 90°, 72°, 108°, 90°. If sector 90° corresponds to 180 units, total = 180 × (360/90) = 720 units.

Example 4 — Histogram
If histogram bin [10–20) has height 8 (frequency 8), midpoint method estimate for mean uses bin midpoints × frequency; for grouped-data questions use cumulative frequencies.

============================================================
7. TACTICAL SHORTCUTS FOR SPEED

- Use approximate reads for elimination: if a bar is clearly between 30 and 40k, test options accordingly.
- Convert all values to simple units (thousands) so arithmetic is easier.
- For percentage comparisons use ratios to cancel totals.
- Precompute cumulative sums if many queries reference the same total.

============================================================
8. PRACTICE PROBLEMS

1) Bar chart shows employees in departments: A=120, B=200, C=160. What % of total are in B? Ans: 200/480 ×100 = 41.67%.
2) Line chart: revenue Q1=50, Q2=60, Q3=55, Q4=70. What is overall % growth from Q1 to Q4? Ans: (70−50)/50 = 40%.
3) Pie chart: sectors 144°, 90°, 72°, 54°. If smallest sector = 54° corresponds to 81 items, find total. Ans: 81 corresponds to 54/360=0.15 ⇒ total=81/0.15=540.
4) Histogram with bins [0–10):5, [10–20):10, [20–30):5. Estimate mean using midpoints. Ans: compute (5×5 +10×15+5×25)/20 = (25+150+125)/20 =300/20=15.
5) Combo: compare two pies with different totals — convert sectors to % before comparing.

============================================================
9. COMMON ERRORS & HOW TO AVOID THEM

- Misreading scales: always check tick labels.
- Ignoring legend: stacked/clustered series depend on color coding.
- Mixing units: convert before calculations.
- Rounding early: keep exact until final step; only round for answer choices.

============================================================
10. EXTRA PRACTICE (CHALLENGES)

- Given a stacked bar with multiple components, find which component contributes most to overall increase across categories.
- Use scatter plot data to estimate correlation coefficient sign and strength qualitatively.
- Create a small dataset and sketch its histogram; practice reading mean/median/mode from grouped data.
